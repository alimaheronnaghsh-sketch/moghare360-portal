# Product UX Scope

## Phase
M31–M37: Product UX + Soft Run Interface Layer

## Mission 31 Scope (In)
- Design tokens (colors, spacing, typography, shadows, radius, z-index)
- Component style primitives (buttons, cards, badges, forms, tables, alerts, KPI, timeline, empty state, diagnostic)
- Layout system (app shell placeholder, grid, responsive)
- RTL Persian typography rules
- Static HTML demo page (no PHP, no DB)

## Mission 31 Scope (Out)
- Login production change
- Auth architecture change
- Permission model change
- Core database schema change
- Tenant / multi-company / SaaS
- Legacy portal change
- SQL execution
- PHP operational workflow change
- Write logic change
- Production deploy

## Target Users
Internal shop staff — Soft Run prototype environment

## Final Scope Decision
Mission 31 = visual DNA only; no backend or auth impact.
