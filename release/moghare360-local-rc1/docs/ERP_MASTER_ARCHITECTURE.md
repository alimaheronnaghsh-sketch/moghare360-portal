عنوان:
MOGHARE360 ERP Master Architecture - Intake To Delivery

نقش پروژه:
این پروژه باید به یک ERP کامل برای مرکز خدمات تخصصی خودرو تبدیل شود.

هدف اصلی:
انتقال فکر، کنترل، تصمیم‌گیری، پیگیری، آموزش، گزارش‌گیری و کاهش خطای انسانی از افراد به سیستم.

الزامات اصلی:

1. سیستم باید نقش‌محور باشد.
هر کاربر فقط چیزهایی را ببیند که به نقش او مربوط است.

2. سیستم باید فرآیندمحور باشد.
هر پرونده خودرو فقط یک وضعیت فعال داشته باشد.

3. سیستم باید قابل توسعه باشد.
هیچ فرم یا جدول جدیدی بدون اتصال به ماژول، نقش، وضعیت و خروجی مدیریتی ساخته نشود.

4. سیستم باید ساده باشد.
کاربر کم‌سواد باید بتواند با کمترین کلیک کار کند.

5. سیستم باید ضدجزیره‌ای باشد.
پذیرش، فنی، انبار، خرید، مالی، CRM، مدیریت و ترخیص باید روی یک پرونده واحد کار کنند.

6. سیستم باید دانش‌ساز باشد.
گزارش‌های دیاگ، ایرادات تکراری، خطاهای خاص، قطعات مصرفی، زمان تعمیر و برگشت‌کاری باید برای آموزش آینده تیم فنی استفاده شود.

7. سیستم باید خروجی‌محور باشد.
هر بخش باید خروجی Excel, Word, PDF داشته باشد.

8. سیستم باید ورودی‌پذیر باشد.
در فازهای مختلف باید بتواند از Excel, Word, CSV و API داده بگیرد.

9. سیستم باید AI-ready باشد.
ساختار داده باید طوری باشد که در آینده AI بتواند:
- خطاهای عمومی را از خاص جدا کند
- پیشنهاد عیب‌یابی بدهد
- آموزش فنی بسازد
- داشبورد مدیریتی تحلیل کند
- ریسک تأخیر، هزینه و برگشت‌کاری را هشدار دهد

10. سیستم باید قابلیت اتصال API داشته باشد.
برای اتصال آینده به:
- نرم‌افزار حسابداری
- سامانه پیامک
- کاتالوگ قطعات
- فروشگاه قطعه
- CRM
- سیستم حضور و غیاب
- گزارش‌های دیاگ
- ابزارهای AI
